# Skill: QA Test Pass

Use after implementation or before release.

## Procedure

1. Read PRD acceptance criteria and task evidence.
2. Run install/lint/typecheck/test/build/browser checks as applicable.
3. Test happy paths and important failure paths.
4. Add defects to `docs/TASK_QUEUE.md` with severity and reproduction steps.
5. Do not mark release ready if high-severity defects remain.

## Verification

QA handoff includes commands, exit codes, flows tested, defects, and pass/fail recommendation.
