# Skill: Create PRD

Use when starting a project or when requirements changed.

## Procedure

1. Read `docs/PROJECT_BRIEF.md`.
2. Create/update `docs/PRD.md` with summary, goals, non-goals, users, pages/features, acceptance criteria, and open questions.
3. Do not invent unknown business facts; list them as questions.
4. Add implementation tasks to `docs/TASK_QUEUE.md` only after acceptance criteria exist.

## Verification

PRD must be specific enough that a frontend/backend agent can implement without guessing major product decisions.
