# Skill: Architecture Plan

Use before implementation begins or when stack/integration choices change.

## Procedure

1. Read PRD and project constraints.
2. Create/update `docs/ARCHITECTURE.md` with stack, repo structure, routes, data model, APIs/integrations, env vars, security boundaries, deployment target, and validation contract.
3. Prefer simple, boring architecture that agents can safely modify.
4. Add risks and required human decisions.

## Verification

Architecture includes commands/checks needed to prove work is done.
