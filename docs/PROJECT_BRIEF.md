# Project Brief

## Client / Project

- Client: Mobili Top
- Project name: Mobili Top production catalog website
- Business goal: Convert visitors into WhatsApp leads for mattresses, bases, pillows, protectors, furniture, and promotions while keeping an admin-manageable catalog.
- Target users: Venezuelan customers shopping for sleep products who need fast guidance by WhatsApp before buying.
- Success metric: More qualified WhatsApp inquiries and catalog/product views with a trustworthy mobile-first experience.

## Scope

- Pages/screens needed: Home, products catalog, category pages, brand pages, promotions, product detail, about, warranty, contact, admin panel.
- Core features: catalog browsing, filters, product detail, WhatsApp CTAs, lead capture, site settings, admin CRUD, SEO routes, PWA/mobile shell.
- Integrations: WhatsApp links, PostgreSQL/Prisma, NextAuth admin, Docker/Caddy deployment.
- Content/assets available: current seed data, brand name, existing catalog structure.
- Content/assets missing: final real product photos, real phone/address/social URLs, final domain, production admin credentials.

## Constraints

- Stack preference: Existing Next.js App Router + TypeScript + Prisma + PostgreSQL + Tailwind CSS.
- Deployment target: Docker Compose with app + Postgres + Caddy or equivalent VPS deployment.
- Budget/time constraints: Build from existing codebase; avoid risky rewrites.
- Brand rules: Premium but accessible rest/comfort brand; Spanish copy; WhatsApp-first commerce.
- Legal/compliance constraints: Do not invent warranty terms beyond “según producto/fabricante”; avoid committing secrets.

## Human Decisions Needed

- [ ] Confirm real WhatsApp number, address, email, Instagram/Facebook/TikTok, domain, and final admin credentials.
- [ ] Confirm exact warranty/payment/delivery policies before launch.
- [ ] Provide real product photos or approve current generated visual placeholders.
